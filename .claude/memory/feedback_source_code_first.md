讲解一个新概念时，必须先展示相关的源代码，再解释这段代码的含义和背后的概念。读者先看到源码，知道"为什么要讲这个"，然后才能理解讲解内容。

**Why:** 不先展示源码，读者不知道为什么要突然讲解这个概念、这个概念从哪来的、跟当前项目有什么关系。源码是第一手材料，讲解是辅助理解源码的，不能本末倒置。

**How to apply:**
- 引入新概念的章节，第一步是展示涉及该概念的源码片段
- 然后再解释"这段代码在做什么"、"涉及什么概念"
- 概念讲解始终围绕源码展开，不能脱离源码空谈理论
