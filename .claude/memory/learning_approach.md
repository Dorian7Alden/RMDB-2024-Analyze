`db2026-x/` 是空的题目框架（竞赛选手拿到的初始代码），`src/` 是完整的参考实现。

## 学习方法

- 学习重点是框架中**标注 TODO** 或**方法体为空**（返回 nullptr / 什么都不做）的待实现部分
- 讲解时对比 `db2026-x/`（要做什么）和 `src/`（怎么做），展示框架的空实现 → 参考的完整实现
- 框架中已经完整实现的内容（如 parser、SM 管理器等）只做简要介绍，不深入

## 关键原则

- **绝不把框架的未完成代码当成"设计如此"来强行解释**——空函数就是没实现，不是"简化"
- **绝不把框架的不完整逻辑说成是"设计选择"**——TODO 就是 TODO
- 框架展示了"问题"（哪里需要实现），参考实现展示了"答案"（正确的做法），两者对照学习

## 待实现内容识别方法

对比两个目录的差异：
- `diff db2026-x/src/<file> src/<file>` 查看具体方法体差异
- 框架文件中 `beginTuple() {}`、`nextTuple() {}`、`Next() { return nullptr; }` 这类空方法即为学习目标
- 框架中不存在的文件（如 `executor_aggregate.h`）也是需要实现的新模块

Why: 竞赛框架本身就是设计好的学习路径，空方法标注了"需要掌握的核心知识点"。
How to apply: 每一章教程先确认框架中对应模块的待实现内容，再对比参考实现进行讲解。
