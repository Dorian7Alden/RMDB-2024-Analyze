对比框架 `db2026-x/` 和参考实现 `src/` 时，不能只列出差异清单。必须给出对比分析：

- **哪些是关键变化**：区分核心改动和次要细节
- **为什么这么改**：说明背后的考虑（性能？并发？简化？扩展性？）
- **改了什么 + 为什么**：每个变化都要有原因，而不是只陈述"A 变成了 B"
- **别有用心处理的细节**：指出参考实现中那些经过深思熟虑的细节设计

这样才能真正讲懂，让读者理解设计决策而非死记代码差异。

**Why:** 只列差异等于让读者自己猜原因，初学者没有能力反推设计意图。讲清楚"为什么"才能培养设计思维。

**How to apply:** 每个"框架 vs 参考实现"对比表格后，必须跟一段分析文字，挑出最关键的变化讲透原因。次要细节可以一笔带过。
