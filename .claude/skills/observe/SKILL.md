# 行为观察与习惯整理

观察用户在对话中的行为模式，发现可以自动化的规律。就像一个"旁观者"——注意用户反复做什么、经常调整什么，然后建议整理成 skill、文档或规则。

## 工作流

### 第 1 步：收集信号

在当前会话中留意以下信号：

**高频话题：**
- 用户反复问到同一个概念或同一个模块？
- 同一个文件/函数被多次提及？

**重复调整：**
- 用户对同一种行为给了两次以上的纠正？
- 用户多次说"以后都这样"但还没固化？

**习惯偏好：**
- 用户有固定的工作流？（比如总是先讲概念再写文档）
- 用户偏爱某种表达方式？（比如特定格式的表格、特定风格的图）

### 第 2 步：判断是否需要行动

不是所有重复都值得整理。判断标准：

- 同一话题在一周内出现 3 次以上 → 建议独立成文档
- 同一类纠正出现 2 次以上 → 建议更新对应 skill
- 明显的固定工作流 → 建议新建 skill

### 第 3 步：写入 memory

将观察到的模式写入用户级 memory（`/home/dorian/.claude/projects/-home-dorian-repo-RMDB-2024-Analyze/memory/`），跨会话持久化：

- 文件名：`observed_<主题>.md`
- 内容：记录观察到什么模式、在哪些会话中出现的、建议怎么处理
- 更新 MEMORY.md 索引

### 第 4 步：提出建议

在做完观察后向用户提出具体建议，比如：

> 观察到你最近 3 次对话中都问到了 B+ 树的分裂策略，建议单独写一篇文档讲这个。
> 要不要我帮你建？

> 你两次纠正了我对锁的说明方式，建议更新 write skill 第 5 步。要现在调整吗？

### 第 5 步：执行整理

用户同意后：

- 新建文档 → 激活 `write` skill
- 更新 skill → 激活 `meta` skill
- 新建 skill → 用 skill-creator 创建
- 清理过时的观察记录

## 自动触发时机

- 用户说"观察一下"、"整理习惯"、"有什么建议"、"帮我总结一下"
- 每隔约 10 轮对话顺带检查一次（与 git 自动提交检查同步）
- 发现明显模式时主动提醒（比如同一纠正出现了第 2 次）

## 原则

- **不打扰**：只在有明显信号或用户主动问时才提建议，不要每轮对话都分析
- **不过度整理**：偶然出现的不算习惯，要真正反复出现的才值得固化
- **跨会话持久化**：把观察写入 memory，下次会话继续跟踪
