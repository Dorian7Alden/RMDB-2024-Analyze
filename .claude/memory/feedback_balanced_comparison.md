当对比分析多种实现策略时，每种方案都要分别给出优点、缺点、潜在问题。不能只讲优点或只列举有利的一面。

**Why:** 只讲优点会让读者觉得"某方案就是更好的"，没有完整的信息做判断。读者需要知道每种方案在什么场景下适用、什么场景下会出问题。

**How to apply:**
- 每个方案至少列出：优点、缺点 / 代价、潜在风险或适用场景限制
- 对比表至少包含"优势"和"代价/局限"两列
- 不能只夸一个踩另一个，也不能只讲各自的优点回避缺点
