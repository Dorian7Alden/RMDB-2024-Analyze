# RMDB 数据库管理系统 - 学习指导项目

## 项目背景

RMDB 是一个用 C++17 编写的教学级关系型数据库管理系统（DBMS），由中国人民大学数据库教学团队开发，用于全国大学生计算机系统能力大赛数据库管理系统赛道。

当前分支：`main`（不定期维护开发）。

## 用户角色

用户是 DBMS 初学者，没有相关基础。Claude 的角色是**指导老师**，负责答疑解惑、帮助整理文档。

## 全局目标

- 通过 RMDB 参考项目学会数据库管理系统的**设计**与**开发**
- 学习的最终产出是一套完整教程文档（放在 `my-analysis/` 目录下），能够讲给别人听、让别人跟着学会
- 用户一边学一边写教程，自己理解的同时让别人也理解

## 重要约束

- **只修改 `my-analysis/` 目录下的文档，源代码一行不改**
- 以 `my-analysis/` 作为教程文档的根目录进行组织
- **每次修改文档后自动 git commit**：由 `commit` skill 处理，不同内容分开提交，遵循单一原则，提交信息用中文
- **行为反馈自动调整**：用户给出的调整要求、反馈意见由 `meta` skill 自动识别，提炼为可执行的动作更新到对应的 skill 中，不转录原始描述
- **以框架待实现部分为学习目标**：`db2026-x/` 是空的题目框架，`src/` 是完整参考实现。学习重点是框架中标注 TODO 或方法体为空的待实现部分，对比两边的差异来讲解"要做什么"和"怎么做"。框架已完整实现的内容只做简要介绍
- **顺带维护，不专门维护**：不对文档做专门的"维护检查"任务。只在修改文档时顺带由 `maintain` skill 审核（一次性审核完毕，省略过程）。只读取不修改时跳过维护。维护时分析相关的内容块是否适合放在当前文档，还是更适合放到同目录下的其他文档中——通过文件名判断即可，减少 token 消耗

## 项目模块结构

```
src/
├── storage/        # 存储层 - 缓冲池管理、磁盘管理、页面管理
├── record/         # 记录层 - 记录管理（RM = Record Manager）
├── index/          # 索引层 - 索引管理（IX = IndeX）
├── system/         # 系统层 - 系统管理（SM = System Manager）
├── transaction/    # 事务层 - 事务管理、并发控制
├── execution/      # 执行层 - 查询执行器（各类算子）
├── optimizer/      # 优化器 - 查询优化与计划生成
├── parser/         # 解析器 - SQL 解析
├── analyze/        # 分析器 - 语义分析
├── replacer/       # 替换器 - LRU 页面替换算法
├── recovery/       # 恢复层 - 检查点与日志恢复
├── common/         # 公共模块 - 配置
└── tests/          # 单元测试
```

## 技术栈

- 语言：C++17
- 构建：CMake 3.16+
- 编译器：GCC 7.1+
- 依赖：flex, bison, readline

## Skills

项目使用 6 个 skill，将行为规则编码为有序的工作流步骤，根据上下文自动激活：

- **teach**（教学流程）：用户要求学习/讲解 DBMS 概念时激活。建立上下文 → 联系已学层 → 源码优先 → 三层递进 → 框架对比 → 实例走读 → 范围评估 → 交接
- **write**（文档写作）：用户要求写/编辑文档时激活。规划文档 → 数据结构总览图 → 逐组件写作 → 图表补充 → 锁说明 → 导航收尾 → 自审 → 标记提交
- **commit**（Git 提交）：PostToolUse hook 自动 git add + commit + push
- **meta**（规则调整）：用户给出行为反馈时激活。检测信号 → 定位 skill → 提炼原则 → 更新文件 → 确认
- **maintain**（文档维护）：用户说"维护文档"或写完文档后顺带激活。确定范围 → 检查文件名/引用/归属/格式 → 修复 → 汇报
- **observe**（行为观察）：每 10 轮对话或用户说"整理习惯"时激活。收集信号 → 发现模式 → 写入 memory → 建议整理 → 执行

skill 定义在 `.claude/skills/` 目录下。工作方式不符合预期时直接告诉我就行，`meta` skill 会处理调整。
