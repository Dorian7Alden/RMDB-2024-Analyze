# 文档维护

维护 `my-analysis/` 下的教程文档。只检查最近查看和直接关联的文档，不全量扫描。

## 工作流

### 第 1 步：确定维护范围

- 用户指定了文件 → 只检查那些文件
- 用户未指定 → 检查当前打开的文件 + 其导航链接指向的相邻文档
- 不扫描整个 `my-analysis/` 目录

### 第 2 步：逐文件检查

对每个范围内的文件，依次检查以下项目：

**文件名与内容匹配：**
- 文件名中的话题前缀是否反映文档实际覆盖范围？
- 如果文档内容已经超出文件名范围很多 → 建议拆分或改名

**交叉引用有效：**
- 文档内引用的文件路径是否存在？
- 导航链接（上一节/下一节）指向的文件是否存在？
- 引用的源码路径和行号是否仍然准确？

**内容归属合理：**
- 有没有内容块更适合放在同目录的其他文档中？
- 有没有某个主题展开过多、应该独立成文的？
- 判断方式：通过文件名判断各文档的职责范围，不需要精读每个文档

**格式一致性：**
- 是否遵循一句一段？
- 是否所有解释性段落有指示性标签开头？
- 首次出现的缩写是否给了全称？
- 末尾是否有导航链接？

**内容质量检查（write skill 第 3/4/5 步 + teach skill 第 4 步）：**

教学原则（讲解型文档）：
- 是否有三层递进结构（概览 → 逐块拆解 → 串联回顾）？
- 关键概念是否遵循"先是什么、再为什么、最后在哪里用"的顺序？
- 是否有框架对比（db2026-x vs src）？
- 是否有具体实例走读？
- 抽象概念是否配了实例？
- 是否联系了已学层的相似处和差异处？

写作规范（所有文档）：
- 是否一句一段？
- 每个解释性段落是否有指示性标签（含义/作用/场景/源码/示例/输入/输出）开头？
- 缩写首次出现是否给了全称？
- 术语是否严格区分——"键"（key）vs"键值对"（key-value pair），禁止单独说"键值"？
- 提到锁时是否说明了全部四个维度（级别/范围/类型/生命周期）？禁止笼统说"这里加锁"。
- 标题是否只含主题词、无破折号括号英文翻译？
- Mermaid 图中是否禁用了 `()` `*` `/` 等特殊字符？尖括号是否只用于 Mermaid 支持的 `<br/>` 换行标签，而不是用于包裹文本内容？是否需要图例分离？
- 代码块第一行是否有 `// src/xxx:行号` 注释？代码块外不应重复贴 `**源码**：` 标签。

可读性（所有文档）：
- 文档是否自包含——不需先读三篇别的才能理解这一篇？
- 是否零基础友好——没有突然出现的未解释概念？
- 整体排版是否达到"随时能出版"的标准？

### 第 2.5 步：新人视角模拟

（在逐文件机械检查完毕后、修复前执行）

启动一个 agent（subagent_type=general-purpose），让它以 **DBMS 初学者**的身份阅读范围内的每个文档，对任何感到困惑的地方提出具体问题。Agent 的提问角度：

- 这个概念突然出现，前面没解释过，它是什么？
- 这段描述太抽象了，有没有具体例子？
- 这里为什么需要这个步骤？逻辑上跳了一步，中间发生了什么？
- 这个术语和前面某个术语是同一个东西还是不同的？
- 这段代码/流程看完还是不知道它到底做了什么，能用一个实例走一遍吗？

Agent 的 prompt 要点：
- 明确告知这是教学文档，目标读者是 DBMS 零基础新手
- 提供要检查的文件列表
- 要求逐文件阅读，站在初学者角度提问
- 每个问题要指出具体的文档位置和困惑点
- 不需要修复建议，只需要提问

Agent 返回后，逐个评估其提出的问题：
- 确实会造成新人困惑的 → 立即修改文档
- 新人顺着前面文档读下来应该能理解的 → 简要说明不修改的原因
- 超出本章范围、属于后续章节内容的 → 确认文档已有"后续章节会讲到"的提示

### 第 3 步：修复并提交

- 能直接修的问题直接修（比如更新路径、补充缺失的导航链接）
- 需要用户决策的提出来（比如"这个文档要不要拆成两个？"）
- 修复完成后激活 `commit` skill 提交

### 第 4 步：汇报

简要告诉用户：
- 检查了几个文件
- 修复了哪些问题
- 有什么需要用户决策的事项

## 原则

- **质量第一**：格式问题和内容问题同等对待，发现问题直接修复
- **聚焦最近查看**：不全量扫描，只看用户正在用的文档
- **一次性审核完毕**：省略中间过程，直接给结果
- **文件名即职责**：通过文件名判断内容归属，减少 token 消耗
- **原则引用**：内容质量以 `write` skill 第 3/4/5 步和 `teach` skill 第 4 步为检查标准
